# Web Development Learning Guide

Ek self-paced roadmap web development seekhne ke liye — beginner se job-ready tak.
Isko apni progress track karne ke liye bhi use kar sakte ho (checkboxes tick karte jao).

## How to Use This Guide

- Har section ko order mein follow karo — inka structure ek dusre pe build hota hai
- Har topic ke baad ek chhota project banao, sirf theory mat padho
- Roz thoda time do (30 min bhi ho toh consistency rakho)

---

## 1. Foundations

- [ ] HTML — semantic tags, forms, tables, accessibility basics
- [ ] CSS — box model, flexbox, grid, responsive design (media queries)
- [ ] Git & GitHub — init, add, commit, push, branches, pull requests

**Mini project**: Personal portfolio page (static, no JS yet)

---

## 2. JavaScript Fundamentals

- [ ] Variables, data types, operators
- [ ] Functions, scope, closures
- [ ] Arrays & objects, array methods (map, filter, reduce)
- [ ] DOM manipulation & events
- [ ] ES6+ — arrow functions, destructuring, template literals, spread/rest
- [ ] Async JS — promises, async/await, fetch API

**Mini project**: To-do list app, weather app (using a free API)

---

## 3. Frontend Framework

Pick one (React is most in-demand right now):

- [ ] Components, props, state
- [ ] Hooks — useState, useEffect, useContext
- [ ] Routing (React Router)
- [ ] State management basics (Context API / Redux)
- [ ] Styling — Tailwind CSS or CSS Modules

**Mini project**: Multi-page app (e.g. movie search app, notes app)

---

## 4. Backend Development

- [ ] Node.js basics — modules, npm, event loop
- [ ] Express.js — routing, middleware, REST APIs
- [ ] Authentication — JWT or session-based auth
- [ ] Error handling & validation

**Mini project**: A REST API with CRUD endpoints

---

## 5. Databases

- [ ] SQL — PostgreSQL or MySQL (queries, joins, relations)
- [ ] NoSQL — MongoDB (documents, collections, aggregation basics)
- [ ] Database design — schema design, normalization

**Mini project**: Connect your backend API to a real database

---

## 6. Full Stack Integration

- [ ] Connect frontend to backend (fetch/axios)
- [ ] Full CRUD application with auth
- [ ] Environment variables & config management
- [ ] Basic testing (Jest for unit tests)

**Capstone project**: A complete full stack app — e.g. blog platform,
e-commerce store, or task manager with login

---

## 7. Deployment & DevOps Basics

- [ ] Hosting frontend — Vercel, Netlify, or GitHub Pages
- [ ] Hosting backend — Render, Railway
- [ ] Docker basics (optional but valuable)
- [ ] CI/CD basics (GitHub Actions)

---

## 8. Extra Skills (Ongoing, Post-Basics)

- [ ] TypeScript
- [ ] System design basics (scalability, caching, load balancing)
- [ ] GraphQL
- [ ] AI API integration (using LLMs in apps)
- [ ] Data Structures & Algorithms (for interviews)

---

## Suggested Stack

If unsure where to start, follow this well-documented combo:

**MERN Stack** = MongoDB + Express.js + React + Node.js

## Portfolio Checklist (Before Applying for Jobs)

- [ ] 3–4 solid projects on GitHub, each with a clear README
- [ ] At least one full stack project with authentication
- [ ] Clean commit history — consistent, meaningful commit messages
- [ ] A deployed portfolio site linking to your live projects
- [ ] Resume highlighting projects, not just listed technologies

---

## Notes

- Timeline: roughly **6–8 months** at 2–3 hours/day, consistently
- Don't wait to "finish learning" before building — build while learning
- Getting stuck is normal — searching, reading docs, and asking for help
  (from people or AI tools) is part of the actual skill, not a shortcut
